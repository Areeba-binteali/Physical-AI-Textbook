This document outlines the key entities and concepts that form the "data model" of Module 3, representing the core information that students are expected to learn and understand. It focuses on the definitional and relational aspects of these concepts as they are presented in the textbook.

## Entities

### NVIDIA Isaac Platform
- **Description**: A comprehensive robotics platform comprising simulation (Isaac Sim) and robotics software (Isaac ROS) for accelerating AI-powered robot development.
- **Key Attributes**:
    - `Components`: Isaac Sim, Isaac ROS
    - `Purpose`: Accelerating AI-powered robot development, Physical AI, embodied intelligence
- **Relationships**:
    - `Encompasses`: Isaac Sim, Isaac ROS

### Isaac Sim
- **Description**: A scalable, cloud-native simulation platform for developing, testing, and managing AI robots.
- **Key Attributes**:
    - `Functionality`: Photorealistic simulation, synthetic data generation, virtual environment for development and testing
    - `Characteristics`: Scalable, cloud-native
- **Relationships**:
    - `Part of`: NVIDIA Isaac Platform
    - `Generates`: Synthetic Data

### Isaac ROS
- **Description**: A collection of hardware-accelerated ROS packages for robotics applications, particularly perception.
- **Key Attributes**:
    - `Functionality`: Hardware-accelerated perception pipelines, camera and depth processing
    - `Characteristics`: ROS packages, hardware-accelerated
- **Relationships**:
    - `Part of`: NVIDIA Isaac Platform
    - `Utilizes`: AI Perception Pipelines

### Synthetic Data
- **Description**: Artificially generated data used to train AI models, often from photorealistic simulations.
- **Key Attributes**:
    - `Source`: Photorealistic simulations (e.g., Isaac Sim)
    - `Purpose`: Training AI models
- **Relationships**:
    - `Generated by`: Isaac Sim
    - `Enhanced by`: Domain Randomization
    - `Used for`: AI Perception Pipelines

### Domain Randomization
- **Description**: A technique used in synthetic data generation to vary environmental parameters to improve model generalization.
- **Key Attributes**:
    - `Purpose`: Improve model generalization, prevent overfitting to simulation
    - `Application`: Synthetic data generation
- **Relationships**:
    - `Applied to`: Synthetic Data

### AI Perception Pipelines
- **Description**: Hardware-accelerated software stacks for processing sensor data (e.g., camera, depth) using AI models.
- **Key Attributes**:
    - `Components`: AI models, sensor data (camera, depth)
    - `Characteristics`: Hardware-accelerated
- **Relationships**:
    - `Implemented via`: Isaac ROS
    - `Processes`: Sensor Data
    - `Outputs`: Data for VSLAM, Localization

### Visual SLAM (VSLAM)
- **Description**: A process by which a robot simultaneously estimates its own motion and builds a map of its surroundings using visual sensor data.
- **Key Attributes**:
    - `Functionality`: Simultaneous Localization and Mapping, motion estimation, map building
    - `Input`: Visual sensor data
- **Relationships**:
    - `Utilizes output from`: AI Perception Pipelines
    - `Supports`: Localization, Navigation & Path Planning

### Localization
- **Description**: The process of determining a robot's position and orientation within a map.
- **Key Attributes**:
    - `Functionality`: Position and orientation determination
    - `Input`: Map, sensor data
- **Relationships**:
    - `Supported by`: VSLAM
    - `Prerequisite for`: Navigation & Path Planning

### Navigation2 (Nav2)
- **Description**: A flexible ROS 2 framework for autonomous robot navigation.
- **Key Attributes**:
    - `Framework`: ROS 2
    - `Purpose`: Autonomous robot navigation, path planning
- **Relationships**:
    - `Utilizes`: Localization, VSLAM
    - `Applies to`: Bipedal Humanoid Robotics (among others)

### Bipedal Humanoid Robotics
- **Description**: Robotics focusing on two-legged human-like robots, posing unique challenges for balance and locomotion.
- **Key Attributes**:
    - `Characteristics`: Two-legged, human-like
    - `Challenges`: Balance, locomotion, dynamic environment interaction, physical constraints
- **Relationships**:
    - `Context for`: Navigation & Path Planning (challenges)

### Sim-to-Real
- **Description**: The process of transferring models or policies trained in simulation to real-world robotic systems.
- **Key Attributes**:
    - `Process`: Transfer of learned behaviors from simulation to real world
    - `Challenges`: Overfitting to simulation
- **Relationships**:
    - `Involves`: Simulation (e.g., Isaac Sim), Real-world robotics
    - `Addressed by`: Mitigation strategies (e.g., Domain Randomization)

### Overfitting (to Simulation)
- **Description**: A phenomenon where an AI model performs well in simulation but poorly in the real world due to learning simulation-specific artifacts.
- **Key Attributes**:
    - `Cause`: Learning simulation-specific artifacts
    - `Effect`: Poor real-world performance
- **Relationships**:
    - `A challenge in`: Sim-to-Real
    - `Mitigated by`: Domain Randomization, other Sim-to-Real strategies
