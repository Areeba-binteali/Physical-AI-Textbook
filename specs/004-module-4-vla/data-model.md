## 1. Overview

This document defines the core data structures that will be used to pass information between the nodes in the Vision-Language-Action (VLA) pipeline. These are not database models, but rather in-memory representations and message structures.

## 2. Core Entities

### 2.1. `VoiceCommand`

Represents the initial input from the user after transcription.

-   **Description**: A simple structure holding the transcribed text from the user's voice command.
-   **Fields**:
    -   `transcript` (string): The raw text output from the Speech-to-Text service.
-   **Used By**: `STT Node` -> `LLM Planner Node`
-   **ROS 2 Representation**: `std_msgs/String` on a topic (e.g., `/voice_transcript`).

### 2.2. `TaskPlan`

Represents the high-level plan generated by the LLM.

-   **Description**: An ordered sequence of steps that the robot should perform to accomplish the user's goal.
-   **Fields**:
    -   `steps` (string[]): An array of strings, where each string is a natural-language description of an action (e.g., "Navigate to the kitchen table", "Find the red can").
-   **Used By**: `LLM Planner Node` -> `Orchestrator Node`
-   **ROS 2 Representation**: A custom message or service, likely `vla_interfaces/srv/GeneratePlan.srv` which returns a `string[]`.

### 2.3. `RobotAction`

Represents a single, concrete action that a robot subsystem can execute.

-   **Description**: A structured command sent to a specific ROS 2 Action Server (like Nav2 or a manipulation controller). This is the output of the Orchestrator node's mapping logic.
-   **Fields**:
    -   `action_name` (string): The name of the ROS 2 Action Server to call (e.g., `/navigate_to_pose`).
    -   `action_goal` (varies): The goal message specific to the action server. This could be a `PoseStamped` for navigation, a `joint_trajectory` for manipulation, etc.
-   **Used By**: `Orchestrator Node` -> `Navigation/Manipulation Action Servers`
-   **ROS 2 Representation**: This corresponds to the goal portion of a ROS 2 Action definition (e.g., `nav2_msgs/action/NavigateToPose`).
